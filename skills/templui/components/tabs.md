# Tabs

> A set of layered content panels (tab panels) that are displayed one at a time, controlled by tabbed trigger buttons.

## Install

```bash
templui add tabs
```

Dependencies auto-installed: none

Add to your base layout `<head>`: `@tabs.Script()`

## Update

```bash
templui add tabs
```

To update all: `templui --installed add`

## API

### Enums

No enums.

### Props

```go
type Props struct {
	ID         string
	Class      string
	Attributes templ.Attributes
}

type ListProps struct {
	ID         string
	Class      string
	Attributes templ.Attributes
}

type TriggerProps struct {
	ID         string
	Class      string
	Attributes templ.Attributes
	Value      string
	IsActive   bool
	TabsID     string
}

type ContentProps struct {
	ID         string
	Class      string
	Attributes templ.Attributes
	Value      string
	IsActive   bool
	TabsID     string
}
```

## Components

| Function | Props | Description |
| -------- | ----- | ----------- |
| `Tabs(props ...Props)` | `Props` | Root wrapper `<div>`. Generates a unique internal tabs ID (or uses `Props.ID`) and passes it to children via context. |
| `List(props ...ListProps)` | `ListProps` | Container `<div>` for trigger buttons. Renders the tab bar with muted background and rounded styling. |
| `Trigger(props ...TriggerProps)` | `TriggerProps` | A `<button>` that activates its associated content panel. `Value` is **required** and must match a corresponding `Content` value. Set `IsActive` to mark the default active tab. |
| `Content(props ...ContentProps)` | `ContentProps` | A `<div>` panel shown when its matching trigger is active. `Value` is **required** and must match a corresponding `Trigger` value. Set `IsActive` to display by default. |
| `Script()` | — | Outputs the `<script>` tag for tab switching behavior. Auto-generated by `templui add`. Place in `<head>`. |

## Composition

```
Tabs
  List
    Trigger
    Trigger
    ...
  Content
  Content
  ...
```

`List` contains all `Trigger` buttons. `Content` panels are siblings of `List` inside `Tabs`. Each `Trigger` is paired with a `Content` by a matching `Value` string.

## Dependencies

None — only uses `utils` package.

## Context

- `Tabs` stores the tabs ID in context via `context.WithValue(ctx, "tabsId", tabsID)`.
- `List`, `Trigger`, and `Content` retrieve it with `IDFromContext(ctx)`.
- `Trigger` and `Content` accept an explicit `TabsID` prop that overrides the context value, enabling use outside the `Tabs` wrapper.

```go
func IDFromContext(ctx context.Context) string
```

## Data Attributes

| Attribute | Element | Description |
| --------- | ------- | ----------- |
| `data-tui-tabs` | `Tabs` root | Marks the root tabs container. |
| `data-tui-tabs-id` | All elements | Shared ID linking triggers and content panels together. |
| `data-tui-tabs-list` | `List` | Marks the trigger list container. |
| `data-tui-tabs-trigger` | `Trigger` | Marks a tab trigger button. |
| `data-tui-tabs-content` | `Content` | Marks a tab content panel. |
| `data-tui-tabs-value` | `Trigger`, `Content` | The value used to pair a trigger with its content. |
| `data-tui-tabs-state` | `Trigger`, `Content` | `"active"` or `"inactive"`. Drives styling via `data-[tui-tabs-state=active]:*` selectors. |

## JavaScript API

The bundled script exposes a global API:

```js
window.tui.tabs.setActive(tabsId, value)
```

Programmatically switch the active tab by providing the tabs ID and the target trigger/content value.

## Usage

```templ
@tabs.Tabs() {
  @tabs.List() {
    @tabs.Trigger(tabs.TriggerProps{Value: "account", IsActive: true}) {
      Account
    }
    @tabs.Trigger(tabs.TriggerProps{Value: "password"}) {
      Password
    }
  }
  @tabs.Content(tabs.ContentProps{Value: "account", IsActive: true}) {
    <p>Make changes to your account here.</p>
  }
  @tabs.Content(tabs.ContentProps{Value: "password"}) {
    <p>Change your password here.</p>
  }
}
```
